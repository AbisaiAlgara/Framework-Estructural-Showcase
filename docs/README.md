# Documentación técnica

Esta carpeta contiene documentos de apoyo para explicar la arquitectura, criterios de modelado y convenciones internas del framework estructural.

## Contenido

* **Arquitectura del optimizador:** descripción general de módulos, flujo de datos, responsabilidades internas y estructura del sistema.
* **UML de arquitectura:** diagramas de clases y relaciones principales entre componentes del framework.
* **Convención de orientación beta:** criterios para la orientación local de perfiles IR/OR y su interpretación estructural.
* **Figuras auxiliares:** esquemas o imágenes utilizadas para documentar criterios de modelado, orientación y validación.

## Propósito

El objetivo de esta documentación es demostrar que el framework no se limita a ejecutar análisis aislados, sino que sigue una arquitectura organizada, trazable y extensible para análisis estructural computacional, diseño LRFD y optimización.
